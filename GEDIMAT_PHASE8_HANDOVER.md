## Critical Constraints for Cloud Sessions

‚ö†Ô∏è **IMPORTANT: Claude Code Cloud Limitations**

**What Cloud CAN access:**
- ‚úÖ GitHub raw content URLs (`https://raw.githubusercontent.com/...`)
- ‚úÖ Public URLs (https://...)
- ‚úÖ Files generated within cloud session

**What Cloud CANNOT access:**
- ‚ùå WSL filesystem (`/home/setup/...`)
- ‚ùå Local files on user's machine
- ‚ùå Windows Downloads folder (`/mnt/c/users/setup/downloads/`)
- ‚ùå Git repositories not on GitHub public

**Best Practice for Cloud Prompts:**
1. **Always use GitHub raw URLs** for reference documents
2. **Include all code/specifications inline** in cloud prompt
3. **No file path references** to local WSL/Windows locations
4. **Verify all URLs** are `githubusercontent.com` before deployment

**Example (CORRECT):**
```
Read https://raw.githubusercontent.com/dannystocker/infrafabric/gedimat-evidence-final/intelligence-tests/gedimat-logistics-fr/GEDIMAT_ARENA_REVIEW_COMPLETE_V3.2.md
```

**Example (WRONG):**
```
Read /home/setup/infrafabric/intelligence-tests/gedimat-logistics-fr/GEDIMAT_ARENA_REVIEW_COMPLETE_V3.2.md
```
