## PAGE 1 : VUE D'ENSEMBLE (2 minutes de lecture)

### Le problème en 30 secondes

**Situation actuelle :**
- Coûts d'affrètement externe (>10t) : 120 000-200 000€/an
- Représente : 50-60% du budget logistique interne total
- Modèle fragmenté : chaque dépôt gérait ses enlèvements isolément
- Résultat : délais incohérents, clients insatisfaits, marges réduites

**Racine du problème :**
Pas de consolidation commandes multi-dépôts vers mêmes fournisseurs.
Exemple : Emeris tuiles → 15t Méru + 5t Gisors = 20t total
- Ancien modèle : 2 enlèvements = 270-330€ + délais différents + mauvaise service
- Nouveau modèle : 1 enlèvement + 1 navette = 380€ + fiabilité 95% + client heureux

---

### La solution en 30 secondes

**Centraliser les enlèvements fournisseurs** :
1. Angelique (coordination) regroupe commandes dépôts avant d'enlever
2. Livrer 1 dépôt principal (urgent ou proche) via affrètement externe
3. Redistribuer autres dépôts via navette interne 2×/semaine (très économe)
4. Ajouter alertes automatiques délais fournisseurs (prévention retards)

**Résultat quantifié :**
- Économie brute : 20-25% coûts affrètement = 25 000-45 000€/an
- Satisfaction client : +20-25 points (95% vs 70%)
- Rétention client additionnelle : +3-5% (valeur 50 000-100 000€/an long terme)
- **NET VALUE 5 ans : +60 000-120 000€ (ROI 150-300%)**

  • Si urgence client risque : chauffeur interne express (même <10t)

RISQUE 2 : Fiabilité navette interne insuffisante (absence chauffeur)
──────────────────────────────────────────────────────────────────────
Probabilité : BASSE
Impact : MOYEN (2-3 cas/an, retard redistribution)

Mitigation :
  ✓ Planifier navettes 2×/sem fixe (mercredis & vendredis)

  ✓ Contrat avec transporteur local pour urgence navette
  ✓ Buffer 1 jour dans planning redistribution

  • Fallback transporteur externe pour navette urgence (coûteux)


RISQUE 3 : Dépôt refuse livraison consolidée (défend "son" volume)
───────────────────────────────────────────────────────────────────
Probabilité : MOYENNE (politique interne)
Impact : MOYEN (blocage décision, retards)

  ✓ Communication PDG : règle est URGENCE CLIENT, pas volume dépôt

  ✓ Incitation KPI : dépôt récompensé si taux service haut
  ✓ Transparence : montrer gains par dépôt (tous bénéficient)

Plan B :
  • Escalade PDG si conflict persistant (rare)
  • Revoir allouées transporteurs selon coopération


RISQUE 4 : Fournisseur retard = alertes inefficace
──────────────────────────────────────────────────
Probabilité : BASSE (alertes = prévention)
Impact : MOYEN (retard client malgré alerte)

  ✓ Audit fournisseurs : scoring fiabilité (remplacer si chronic)

Plan B :
  • Express pickup si problème confirmé
  • Accepter surcoût urgent pour client (valeur > coûts)
