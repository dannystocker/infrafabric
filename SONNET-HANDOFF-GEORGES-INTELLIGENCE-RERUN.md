## 8. IF.TTT COMPLIANCE CHECKLIST

**IF.TTT = Traceable, Transparent, Trustworthy**

This is the InfraFabric standard. Your rerun MUST meet 100% compliance.

### Traceable Checklist

Every claim must link to observable source.

- [ ] **Behavioral frameworks:** Each framework tagged with (Author, Institution, Year, n=X)
  - Example: (Langer, MIT, 2006, n=507) ✓
  - NOT: "research shows" ✗

- [ ] **Financial metrics:** Each number shows its derivation
  - Example: €280K = (€400K × 70%) - €20-50K ✓
  - NOT: "€280,000 annual savings" ✗

- [ ] **Pilot gates:** Each criterion is measurable
  - Example: ">60% of team using platform" (measured via usage logs) ✓
  - NOT: "good adoption" ✗

- [ ] **Operational deadlines:** Each deliverable has specific time
  - Example: "Day 3 (by 15:00 Paris time): Gap analysis document delivered" ✓
  - NOT: "week 1: audit phase" ✗

### Transparent Checklist

Every major assumption must be flagged.

- [ ] **Confidence levels stated** for each major claim
  - Format: [VERIFIED], [ASSUMED], [INFERRED], [PROJECTION], [UNKNOWN]
  - Example: "€400K annual spend [ASSUMED: based on Constructech 2023 n=312 SME average]"

- [ ] **Research limitations acknowledged**
  - Example: "(Constructech 2023 study limited to French BTP sector; IT services may vary 10-15%)"

- [ ] **Unknowns admitted**
  - Example: "Actual team size for Georges varies; formula scales linearly"

- [ ] **Assumptions flagged with [ASSUMPTION] tags** throughout
  - Every major claim that relies on inference should carry this flag

### Trustworthy Checklist

Only primary sources, conflicts addressed, no phantom numbers.

- [ ] **Sources verified as primary**
  - Deloitte 2023 study (published report) ✓
  - "Blog says Deloitte says..." (gray literature) ✗

- [ ] **Conflicting data addressed**
  - Example: "70% waste figure varies 65-85% by source; using mid-range 70% [ASSUMPTION]"
  - NOT: Picking the highest figure without discussion ✗

- [ ] **No phantom numbers**
  - Example: "€100K-€300K partnership value (upper bound: 3 projects/year × €100K; lower bound: 1 project/year × €100K)"
  - NOT: "€100K-€300K" without rationale ✗

- [ ] **Mistakes/corrections documented**
  - Example: "Prior analysis said €280K; updated to €280K = X×Y-Z based on refined assumptions"
  - Format: "(Corrected Nov 22, previous version: X)"
