## 1. ÉNONCÉ DE LA CONTRADICTION

### Dilemme Structurel

La Zone 4 des tensions (Pass 5 - Résolution Contradictions) oppose deux approches incompatibles pour l'optimisation logistique Gedimat :

| Approche | Logique | Investissement | ROI Court Terme | Viabilité 3 ans |
|----------|---------|---|---|---|
| **Excel + Scripts Automation** | Gains rapides, ressources internes | 0 € CAPEX | **304% (12 mois)** | Plateau plateau non-scalable |
| **TMS Sophistiqué** | Scalabilité stratégique, automatisation complète | 8-35 k€/an | 264% (18-24 mois) | **Excellente (25+ sites)** |
| **Dilemme apparent** | Over-engineering vs Sunk Cost | | | |

### Risques Antagoniques

**Risque 1 : Over-Engineering (TMS acheté avant prêt)** → Coûts stranded, ROI 0, rejet équipes
- **Exemple** : Investir 35k€ TMS Sinari en mois 0, puis découvrir données manquantes = TMS paralysé
- **Coût invisible** : 35k€ perdu + surcoûts changement + perte crédibilité IT

**Risque 2 : Sunk Cost Excel (trop longtemps)** → Perte compétitivité, attrition clients, goulot Angélique
- **Exemple** : Rester Excel 36 mois alors que croissance +50% justifie TMS mois 12
- **Coût invisible** : +15-20% surcoûts logistiques, -5-8% attrition clients, expertise Angélique figée

### Citation Tension Pass 5

**SYNTHESE_PLATEAU_ZONES_TENSION_ARBITRAGES.md, Section 4** :
> *« Phase 1 (Mois 1-4) : Excel automation, ROI 304% visible immédiatement
> Phase 3 (Mois 13-36) : Migration Progressive TMS pour scalabilité 2026
> **Approche échelonnée = seule stratégie évitant piège sunk cost**»*

### Enjeu Décisionnel

- **Sans résolution** : Fragmentation décisions (pression Q1 pour Excel vs vision 2026 pour TMS) → confusion, coûts cachés +15-20%
- **Avec résolution** : Roadmap adaptative, transitions déclenchées par métriques objectives, scalabilité assurée, ROI composé

## 2. CADRE THÉORIQUE DEWEY : EXPÉRIMENTALISME PRAGMATIQUE

### Philosophie : Apprendre par Expérience, Pas a Priori

**John Dewey - Expérimentalisme** :
L'intelligence humaine n'est pas théorique mais **pratique et adaptative**. Plutôt que de débattre a priori « Excel vs TMS = meilleur », l'organisation doit **tester hypothèses empiriquement**, mesurer, puis ajuster.

**Application Gedimat** :
1. **Hypothèse Phase 1** : « Excel avec scoring VRP peut générer ROI 300% en 12 mois avec ressources internes »
   - Test empirique : Implémenter Excel 90 jours, mesurer gains réels vs estimés (coûts transport, satisfaction Angélique)
   - Validation terrain > théorie

2. **Décision transition Phase 1→2** : Non basée sur opinion, mais sur **déclencheurs mesurables**
   - SI (volumes +30% ET maintenance Excel >8h/sem) → Passer Phase 2
   - Sinon, optimiser Excel 12 mois de plus

3. **Hypothèse Phase 2** : « API tracking + scripts Python = 50% charge Excel libérée, préparation TMS sans surcoûts »
   - Test : Ajouter tracking temps réel 90 jours, valider avant full TMS

4. **Hypothèse Phase 3** : « TMS devient ROI positif SEULEMENT si volumes >+80% ou 4e dépôt ouvert »
   - Déclencheur objectif : Pas d'achat TMS sur mode ou pression commerciale

### Peirce (Pragmatisme) : Choisir Outil qui Fonctionne Maintenant

**Charles S. Peirce - Pragmatisme** :
« *La vérité d'une idée consiste dans ses conséquences pratiques.* »
Plutôt qu'un débat TMS vs Excel « architecturalement meilleur », la décision repose sur : **Quel outil génère résultats mesurables maintenant ?**

**Application** :
- **Mois 0-12** : Excel fonctionne (ROI 304%) → c'est le bon outil, même si « moins élégant »
- **Mois 12-24** : Excel commence échouer (maintenance >8h/sem) → hybrider Excel + API (nouveau outil « qui fonctionne »)
- **Mois 24-36** : Données accumulées + volumes croissants → TMS devient rentable (déclencheur = conséquences pratiques atteintes)

### Voie du Milieu (Dialectique) : Court + Long Terme

**Contradiction apparente** :
- Thèse : Excel court terme (rapide, 0€)
- Antithèse : TMS long terme (lent, coûteux)
- **Synthèse : Roadmap phasée adaptative**

La synthèse résolve la contradiction en reconnaissant que **chaque approche est optimale à son moment** :
- Excel = optimal Mois 0-12 (données fragmentées, petit volume)
- Hybrid = optimal Mois 12-24 (données enrichies, volumes croissants)
- TMS = optimal Mois 24+ (données consolidées, croissance affirmée)

### 3.2 Répartition Cas d'Usage

**Quand Excel optimal ?**
- Volumes < 500 commandes/mois (source : Pass 5)
- Budget < 5 k€/an (startups, tests)
- Données suffisantes pour arbitrage (sinon TMS paralysé aussi)
- Équipe disponible 4-8h/mois maintenance

**Quand Excel devient sous-optimal ?**
- Volumes > 500 commandes/mois OU maintenance Excel > 8h/semaine
- Plus de 1 person-day par semaine passé à coordination manuelle
- Perte données (crash Excel, dépendance Angélique)
- Demandes automation croissantes (notifications, scoring, reporting)

**Quand TMS nécessaire ?**
- Volumes > 5 000 commandes/mois (croissance 10x depuis démarrage)
- Expansion géographique (4e+ dépôt) = multi-dépôts requiert centralisation
- Data suffisantes pour paramétrage (sinon 3 mois gaspillés)
- ROI négatif Excel (coûts maintenance > bénéfices)

**Conclusion benchmark** : Transition Excel → TMS fait partie du cycle normal croissance PME (500 cdes/mois → 5 000 cdes/mois)

## 4. ROADMAP 3 PHASES : ARCHITECTURE RECOMMANDÉE

### PHASE 1 - QUICK WINS EXCEL (Mois 0-12)

#### Objectifs
1. **Activez gains transport rapides** : Réduction coûts -15-20% (source : Pass 5 validation terrain Emeris)
2. **Validez hypothèses scoring** : Approche multicritère (40% urgence, 30% coût, 20% volume, 10% proximité) fonctionne réellement
3. **Accumulez données** : Historique 12 mois = fondation TMS futures specs
4. **Libérez Angélique partiellement** : Alertes auto détectent 60% cas routiniers, elle garde cas exception

#### Outils Déployés
- **Excel avancé** :
  - Import GeSI (client, commandes, fournisseurs)
  - Formules scoring VRP : IF(urgence>8, "prioritaire", IF(coût<seuil, "optimal",...))
  - Tableaux croisés dynamiques : alertes retards fournisseurs, ruptures stocks
  - Power Query : actualisation données quotidiennes depuis API GeSI

- **Dashboard basique** :
  - Email alertes semi-auto (manuellement vérifiées Angélique) : urgences non livrées demain
  - Rapport hebdo : coûts transport, scoring validation terrain, exceptions
  - Suivi KPI : délais livrables, satisfaction clients (via feedback manuels)

- **Python scripts légers** :
  - Script clustering clients par proximité dépôts (2-3h dev)
  - Calcul coûts réels Médiafret vs estimés (1-2h dev)
  - Export reporting PowerBI (1h dev)

#### Investissement Phase 1
- **CAPEX** : 0 € (Excel, Python = free tools)
- **Ressources internes** :
  - Angélique : 40h (specs VRP, validation terrain)
  - IT/data analyst : 30h (setup Excel, scripts Python, PowerBI base)
  - Management : 10h (validation KPI, métriques succès)
  - **Total 80h ≈ 4 000€ charge interne**
- **Outils** : Microsoft 365 (existant) + Power BI Pro 12€/user/mois (2 users) = **288€/an**
- **Budget externe** : 0 € (aucun SaaS)

- [ ] Mois 3 : Former backup person (responsable dépôt Méru) à 50% capacity
- [ ] Mois 6 : Test dépannage = personne backup prend 1 jour/semaine arbitrage
- [ ] Mois 9 : Plan succession = Angélique + backup peuvent tous deux arbitrer

**Coût mitigation** : 20h interne (formation backup) ≈ 1 000€

### Risque 2 : TMS Acheté Trop Tôt (Sunk Cost)

**Scénario** : Mois 6 achat Sinari 35k€, découverte données incomplètes → TMS paralysé 8 semaines → frustration

**Mitigation (Phase 1→2)** :
- [ ] Mois 12 : Valider **data quality audit** avant signature TMS (répartition poids, coûts réels, historique retards, distances)
- [ ] Déclencheurs objectifs = NON négociables (volumes +30% ET maintenance >8h/sem)
- [ ] RFP process = 4 semaines avant signature (pas achats impulsifs)
- [ ] Essai gratuit TMS 2-4 semaines = test réel avant engagement

**Coût mitigation** :
- Data audit : 10-15 jours HT ≈ 3-5 k€ (source : Pass 5 Lacunes Données)
- RFP process : 40h management ≈ 2 k€
